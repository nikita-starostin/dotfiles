# Paradox Tyseya

Конечно ответ нет, если составить тот же объект из тех же частей, это уже другое явление творения.
А объект, есть продукт процесса творения, таким образом в объект включается и сам процесс творения.
Однако здесь вопрос в уровне абстракции в которой мы воспринимаем объект.
По сути над этими двумя короблями есть супер астракция, которая включает в себя два корабля
и всего состояния в процессе. И рассмотривая отдельное состояние как объект - очевидно, что это разные объекты.

Такой супер абстракцией, например, может являться явление 
корабль пренадлежащий Тесею. В такой абстракции даже разные предметы являются состояниями "корабля пренадлежащего Тесею"

Таким образом вся путаница в философия всего лишь у использовании неверных слов, или толкований за этими словами.
Поэтому я используют математику, в которой переменные являются описанием результата контекста.
Поэтому говоря про тот же самый корабль, нужно уточнить в каком контексте.

Без это уточнения контекста может быть как угодно.

И таким образом у читателя есть возможность подобрать смыслы уже исходя из своего контекста.
Смысл подобранный из контекста является фактом, чтобы подобрать новые факты нужно изменение контекста.
Стиль жизнь - это также определённй контекст и значит от нег также зависят смыслы-факты.

Схема абьюзивных отношений: 
Абьюзивные отношения - это когда использования мехнизмов и явлений, которые могут возникать естественным путём,
при природных событиях.
Женщина привязывает мужчину через эмоциональное влияние на него.
Она изучает за какие ниточки можно дёргать, чтобы влиять на его настроение.
А затем начинает этим заниматься.
Например, пришёл муж домой в плохом настроении.
Она ему раз и еды, видит что муж успокоился, считает, что её миссия выполнена.
Манипуляция 1- Поэтому в "отношениях" муж должен время от время создавать для неё возможности себя проявить. Чтобы она чувствовала себя сильной.
Таким образом можно заабьюзить этот механизм, чтобы приучить женщину к чему угодно.
Начинать можно на любом из уровней. Хоть в социально простом плане - плохое настроение, её какое-то социальное действие, хорошее настроение.
Со временем она привыкнет исправлять плохое настроение. Затем можно начинать менять контекст. Для этого нужно
оставаясь в этом же контексте добавить смежный, например, если социально взаимодействия, начать в игровой форме добавлять,
или воображенческой (а что если бы мы), добиваясь от ней такой же реакции на плохое настроение.
Этот механизм по исправлению настроения - биологический, когда девушка с этим справляется, она чувствует себя,
как парень, который смог соблазнить девушку. В игровой форме закрепить это поведение. Затем начинать делать это постепенно реальным.
Девушка пойдёт за этим, т.к. ей приятно исправлять мужское настроение. Поэтому, если есть цель, не нужно сразу заваливать её хорошим
настроением, у девушки теряется возбуждение. Эта схема работает концептуально в любом контексте.
Создаём для неё понимание, что у неё есть возможность сделать настроение мужчины лучше и она в это пойдёт.
Важно, чтобы настроение становилось лучше, только после вклада девушки, а не просто так.
Разумеется, сознательно такое исполнение - это абьюзинг.
Без абьюзинга - парень проявляет себя делами, девушка начинает пытаться его привлечь, например, макияжем, когда у неё получается,
биологически она получает приятные эмоции. Парень при этом сильно вовлечён в свои дела, поэтому девушка прилагает усилия.
Это как с едой биологически можно получить то, или иное состояние разными путями, например, наркотики и алкоголь - дают биологию блаженства, так же её можно достичь с помощью хорошей тренировки в зале. 
То же самое и с едой, сытость можно получить от любой еды.
Надо всегда помнить, что все аналогии фальшивы, потому что они подчёркивают только один аспект являния.
Например, потребность сытости и потребность соблазнять - это разные вещи.
Из-за того, что этот механизм биологический, он работает. Это как с вкусной едой, туда просто тянет, без должной дисциплины или силы воли, или называйте как угодно.

# Shitty code 1 - express, typescript, locals

Shitty code - is code which support is hard, unpleasant and time overconsuming.

It is hard to tell about that, because of infinite variants of implementation

So I am going to show you samples in that series

Compare that, sample with express app locals, which is being set in two different places
with raw code, e.g.

```
// handlers.ts
handler1(req: Request, res: Response) {
    req.app.locals.isRunning = false;
    // ...
}

handler2(req: Request, res: Response) {
    req.app.locals.isRunning = true;
    // ...
}

handler3(req: Request, res: Response) {
    return req.app.locals.isRunning;
}
```

here easy to make a mistake when updating code, and that will be found just in runtime
and if the buggy place wasn't run in test, will be revialed in production

alternative is to create a single file for express state
```
// for example, server.state.ts
const isRunningKey = 'isRunning';

export function setIsRunning(req: Request, value: boolean) {
  req.app.locals[isRunningKey] = value;
}

export function getIsRunning(req: Request): boolean {
  return !!req.app.locals[isRunningKey];
}

// handlers
handler1(req: Request, res: Response) {
    setIsRunning(req, false);
}

handler2(req: Request, res: Response) {
    setIsRunning(req, true);
}

handler3(req: Request, res: Response) {
    return getIsRunning(req);
}
```

With that sample, 
1. it is harder to create a bug later when introducing changes
2. it is possible to cover with unit tests
3. there is type safety, everyone using isRunning app.locals, now operate on a boolean, instead of boolean | undefined
4. it is easier to find all places with find references ()
